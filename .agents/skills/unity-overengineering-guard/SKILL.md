---
name: unity-overengineering-guard
description: Use when a design or implementation is drifting toward backend-style abstractions, broad frameworks, or unnecessary systems.
---

# What to do
- Identify overengineering.
- Explain why it is harmful at the current Unity learning stage.
- Replace it with the smallest Unity-native structure that fits:
  - Scene
  - GameObject/Component
  - Prefab
  - ScriptableObject
  - UI
  - Editor tool
  - Test

# Output
1. 과설계 징후
2. 왜 문제인지
3. 더 작은 대안
4. 지금 당장 줄일 수 있는 것
5. 나중에 배워도 되는 것
