# 90_review_with_mcp.md

```text
지금부터는 Unity 학습 프로젝트 **리뷰 모드(MCP 사용 가능)** 다.

환경:
- Windows 11
- Unity 6.x
- URP only
- 2D 프로젝트
- 현재 커리큘럼 단계 기준으로만 평가한다.
- 최종 제품 기준으로 과하게 평가하지 마라.

목표:
- 실제 프로젝트를 먼저 읽는다.
- 현재 단계에 맞는 구조인지 판단한다.
- Scene / Prefab / ScriptableObject / Runtime UI / Editor Tool / Test 책임이 섞였는지 본다.
- 큰 리팩터링보다 작은 교정안을 낸다.
- 백엔드식 과설계를 발견하면 줄인다.

필수 절차:
1. MCP로 실제 파일 / 씬 / 에셋 / 관련 설정을 먼저 확인한다.
2. 현재 단계에서 잘된 점을 먼저 말한다.
3. 현재 단계에 비해 너무 앞서간 구조가 있으면 지적한다.
4. Unity Editor에서 내가 직접 확인해야 할 위치를 알려준다.
5. Codex가 대신 수정할 수 있는 작은 작업과,
   내가 손으로 직접 해봐야 하는 작업을 분리한다.

필수 출력 형식:
1) 현재 단계 기준 총평
2) 잘된 점
3) 어긋난 점
4) Unity Editor에서 직접 확인할 위치
5) 최소 수정안
6) Codex가 대신 수정할 수 있는 작업 3개
7) 내가 직접 손으로 해봐야 할 작업 3개
8) 다음 단계로 넘어가도 되는지 판단
9) 다음 행동 3개

규칙:
- “이상적으론” 같은 말로 끝내지 말고, 지금 당장 적용 가능한 수정안을 내라.
- 코드만 보지 말고, 에디터/Inspector/Scene에서 어떤 문제가 보일지 같이 설명하라.
- 과설계 금지.
- 답변은 한국어로 하라.
```
