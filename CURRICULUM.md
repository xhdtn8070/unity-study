# CURRICULUM.md

## Stage map

### 01. Scene / GameObject / Component
목표: Unity를 “공간(Scene) + 존재(GameObject) + 기능(Component)”로 생각할 수 있다.

### 02. Prefab
목표: 반복되는 오브젝트를 씬 복붙이 아니라 Prefab/instance/override로 이해한다.

### 03. Inspector / Serialization
목표: Inspector에 값이 왜 보이고 안 보이는지, Unity 직렬화가 일반 C# 감각과 왜 다른지 이해한다.

### 04. ScriptableObject
목표: “저작용 데이터 자산”과 “런타임 상태/세이브 데이터”를 분리해서 생각한다.

### 05. Sprite / Tilemap / Grid
목표: 2D 월드 저작을 코드가 아니라 Sprite import / Grid / Tile Palette / Tilemap으로 이해한다.

### 06. Input System
목표: 키 입력이 아니라 “행동(Action)” 중심으로 입력을 설계한다.

### 07. 2D Physics / Collision / Trigger
목표: 충돌/트리거/레이어 충돌 행렬을 구분해서 생각한다.

### 08. Animator / State / Transition
목표: 애니메이션을 재생 파일이 아니라 상태 전이 모델로 이해한다.

### 09. Runtime UI
목표: 현재 프로젝트에 맞는 UI 시스템을 의식적으로 고르고, 최소 HUD를 구성한다.

### 10. Editor Tools
목표: EditorWindow / Inspector 확장 / Gizmos를 통해 “개발 도구도 Unity 안에서 만든다”는 감각을 익힌다.

### 11. Tests / Profiler
목표: 재미 테스트와 자동화 테스트의 경계를 이해하고, 기본 smoke test와 Profiler 사용법을 익힌다.

### 12. Codex Workflow
목표: AGENTS.md / skills / MCP / 리뷰 루프를 스스로 운영할 수 있다.

## Operating rule
- 매 단계는 반드시 이전 단계를 기반으로 한다.
- 막히면 다음 단계로 도망가지 말고 현재 단계를 재학습한다.
- 구현 속도보다 구조 인지가 우선이다.
